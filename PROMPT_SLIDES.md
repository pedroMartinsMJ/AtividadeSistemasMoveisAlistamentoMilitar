# Prompt para Criação de Slides - Projeto Alistamento Militar

## Prompt Principal (Copie e cole na IA de slides):

---

**Crie uma apresentação de slides profissional sobre um projeto de melhoria de UX para o site de alistamento militar brasileiro. Use o seguinte conteúdo:**

**Título do Projeto:** Protótipo de Melhoria UX - Alistamento Militar

**Slide 1 - Capa:**
- Título: "Protótipo de Melhoria UX - Alistamento Militar"
- Subtítulo: "Análise e Redesign da Experiência do Usuário"
- Disciplina: Desenvolvimento de Sistemas Móveis
- Data: Março de 2026
- Ícone: 🎖️

**Slide 2 - Problema Identificado:**
- Site original: alistamento.eb.mil.br
- Problemas:
  * Interface desatualizada e confusa
  * Navegação pouco intuitiva
  * Falta de integração com GOV.br
  * Não responsivo para mobile
  * Ausência de feedback visual
  * Formulários longos e cansativos

**Slide 3 - Objetivo do Projeto:**
- Analisar criticamente a UX do site original
- Propor melhorias na jornada do usuário
- Aplicar conceitos de UX/UI e design responsivo
- Criar protótipo de alta fidelidade (6 telas)

**Slide 4 - Solução Proposta:**
- Integração com GOV.br (autenticação única)
- Design moderno e responsivo
- Sistema de sessão persistente
- Dashboard centralizado
- Formulário em etapas (wizard)
- Geolocalização automática
- Central de ajuda (FAQ)
- Acompanhamento visual de status

**Slide 5 - Tecnologias Utilizadas:**
- HTML5 (estrutura semântica)
- Tailwind CSS (estilização)
- JavaScript Vanilla (interatividade)
- Font Awesome (ícones)
- LocalStorage (persistência)
- Geolocation API + Nominatim (geolocalização)

**Slide 6 - Telas do Protótipo:**
1. Tela Inicial - Hero section impactante
2. Login - Integração GOV.br
3. Cadastro - Wizard de 3 etapas
4. Dashboard - Painel central
5. Status - Timeline de progresso
6. Ajuda - FAQ interativo

**Slide 7 - Funcionalidades Destaque:**
- ✅ Banco de dados mock (5 usuários de teste)
- ✅ Validação real de login
- ✅ Geolocalização para preenchimento de endereço
- ✅ Sessão persistente com localStorage
- ✅ Design 100% responsivo
- ✅ Feedback visual (toast notifications)

**Slide 8 - Comparação: Antes vs Depois:**
- Login: Fragmentado → Integrado GOV.br
- Navegação: Confusa → Simplificada
- Formulário: Longo → Em etapas
- Status: Não claro → Timeline visual
- Design: Datado → Moderno
- Mobile: Não responsivo → 100% responsivo

**Slide 9 - Fluxo do Usuário:**
1. Usuário acessa a landing page
2. Clica em "Entrar com GOV.br"
3. Faz login (CPF: 123.456.789-00 / Senha: 123456)
4. É redirecionado para o Dashboard
5. Acessa os serviços disponíveis
6. Realiza cadastro em 3 etapas
7. Acompanha status do alistamento

**Slide 10 - Resultados:**
- 6 telas funcionais criadas
- Design moderno e responsivo
- Integração simulada com GOV.br
- Sistema de autenticação funcional
- Geolocalização implementada
- FAQ interativo com busca

**Slide 11 - Conclusão:**
- Pequenas melhorias de UX transformam a experiência do usuário
- Integração com GOV.br elimina duplicidade de cadastros
- Design moderno aumenta a usabilidade
- Funcionalidades inteligentes simplificam o processo

**Slide 12 - Demonstração:**
- Link do repositório GitHub
- Instruções para executar o protótipo
- Usuários de teste disponíveis

**Estilo dos slides:**
- Cores: Verde militar (#16a34a) e azul (#2563eb)
- Fontes modernas e legíveis
- Ícones visuais para cada conceito
- Gráficos simples para comparações
- Layout limpo e profissional

## Dicas para a IA de Slides:

1. **Use bullet points** para facilitar a leitura
2. **Adicione ícones** visuais (🎖️📋✅🎯)
3. **Máximo 6 bullet points** por slide
4. **Inclua imagens** dos protótipos se possível
5. **Use cores consistentes** (verde e azul)
6. **Adicione transições** suaves entre slides
7. **Inclua gráficos** simples para comparações
8. **Mantenha fonte legível** (mínimo 24pt)
